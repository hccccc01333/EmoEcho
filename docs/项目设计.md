# 项目设计 - 心迹回声桌宠Agent

---

## 一、产品定位

一句话：将“前任skill”从命令式人格复刻升级为“桌面宠物化、可评估、可控边界”的情感 Agent 产品。

目标用户：
- 情感复盘与陪伴需求用户
- 对数字人格与 Agent 工程感兴趣的开发者

---

## 二、系统架构

```text
DesktopPetUI(Tauri+React)
    -> AgentOrchestrator
        -> PersonaEngine
        -> MemoryEngine(SQLite + Vector)
        -> EmotionEngine(E3-Score)
        -> SkillHub
        -> SafetyGuard
    -> LLMProvider(Local/Cloud)
```

---

## 三、关键模块

1. PersonaEngine
   - 投喂资料，生成 `persona_profile.json`
2. MemoryEngine
   - 事实记忆 + 情绪记忆双索引检索
3. EmotionEngine
   - 基于 `E3-Score` 决定回复策略
4. SkillHub
   - 统一技能协议，支持热插拔
5. SafetyGuard
   - 高风险内容降级与终止策略

---

## 四、技术决策

- 桌面端选择 `Tauri`：体积更小、Windows 友好
- 数据层选择 `SQLite`：本地优先、部署简单
- 检索层选择“关键词 + 向量”混合召回
- 模型层支持双路：本地模型（隐私）+ 云模型（质量）

---

## 五、里程碑

- M1：桌宠 UI + 聊天 MVP
- M2：人格构建 + 记忆检索
- M3：情感算法 + Skill
- M4：评估体系 + Demo 发布
