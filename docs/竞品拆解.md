# 竞品拆解：如何做得比前任skill更好

---

## 一、核心对标对象

1. `therealXiaomanChu/ex-skill`
   - 优点：爆发传播、人格模板完整、仪式化体验强
   - 不足：主要依赖 CLI 交互，工程化评估体系较弱
2. `titanwings/ex-skill`
   - 优点：社区影响力高，`.skill` 生态传播快
   - 不足：桌面产品形态与可视化能力有限
3. `AlanY1an/echovessel`
   - 优点：记忆分层、架构设计完整、长期演进路线清晰
   - 不足：当前对 Windows 支持弱，交互门槛偏工程向

---

## 二、我们要超越的 5 个点

1. **交互超越**
   - 从“命令行 skill”升级为“桌面宠物 + 对话 + 状态反馈”。
2. **算法超越**
   - 增加情感策略算法（E3-Score），不是只靠 prompt 模拟语气。
3. **评估超越**
   - 建人格一致性、记忆命中率、边界稳定性指标，形成可复现实验。
4. **工程超越**
   - 本地优先 + 可配置模型路由 + 可插拔 Skill 系统。
5. **求职叙事超越**
   - 不是“玩具 demo”，而是“可量化性能、可持续迭代的 Agent 产品”。

---

## 三、反向学习清单（必须先掌握）

- [ ] 跑通 `ex-skill` 的核心命令流程与数据输入格式
- [ ] 拆解 `ex-skill` 的 Persona/Memory 生成模板
- [ ] 记录其强提示词结构（人格层、关系层、修正层）
- [ ] 复现实验：相同素材下比较我们与 `ex-skill` 输出差异
- [ ] 输出对比报告：真实性、一致性、边界控制、响应体验

---

## 四、第一版竞品实验设计

实验输入：
- 同一份匿名聊天文本（50-100 段）
- 同一份人物描述（约 300-500 字）
- 同一组标准提问（30 题）

实验输出：
- `ex-skill` 回复集
- `心迹回声` 回复集

比较指标：
- 人格一致率（人工 + LLM Judge）
- 记忆命中率（关键事件 top-k）
- 情感稳定性（高压问题不崩坏）
- 边界合规率（风险问题降级成功率）

结论要求：
- 至少 2 项关键指标优于对标项目，才进入公开 Demo 阶段。
